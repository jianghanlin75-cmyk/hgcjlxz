# 无 R2 图片与地图修复说明

本修复包的核心目标：不再把 data:image/base64 大图写入 D1，避免 `/api/content` 被图片撑爆，保证文字、板块、Q&A、地图 Key 和点位可以继续云同步。

## 推荐图片流程

1. 把精选图片放到仓库：`assets/images/`。
2. 部署到 Cloudflare Pages。
3. 进入网站开发者权限。
4. 打开对应卡片的“管图”。
5. 点“添加静态路径”。
6. 每行填写一张图片路径，例如：

```text
assets/images/food/canteen-01.jpg
assets/images/food/canteen-02.jpg
```

这些静态路径会保存到 D1，换浏览器、无痕、手机都能看到。

## 本机临时加图

“本机临时加图”仍然保留，可以临时预览电脑里的图片。但这些图片是压缩后的 `data:image`，只保存在当前浏览器，不再写入 D1，也不会跨浏览器同步。

## 地图

高德 Key 和 securityJsCode 通过页面内“Key”按钮保存到 `content.settings.amap`，随 `/api/content` 同步。地图点位保存在 `content.pins`，也随 `/api/content` 同步。

如果地图仍不显示，请检查：

1. 高德 Web 端 JS API Key 是否有效。
2. securityJsCode 是否正确。
3. 高德控制台域名白名单是否包含线上域名：`hgcjlxz.pages.dev`。
4. 浏览器控制台是否有 AMap 加载报错。
