# 二次修复说明：开发者口令、地图白屏、无 R2 图片

## 1. 换浏览器为什么又让输入开发者口令

旧版把“开发者口令”保存在浏览器 localStorage 里。Edge、夸克、无痕、手机的 localStorage 互不相通，所以换浏览器会像第一次使用一样。

这版已改成：

- 云端同步可用时，不再让新浏览器“重新设置口令”。
- 只提示输入 Cloudflare Pages 环境变量 `ADMIN_TOKEN`。
- 输入后会通过 `/api/verify` 验证。
- 验证成功后，才进入开发者权限。

注意：为了安全，`ADMIN_TOKEN` 不会也不能公开同步给所有浏览器。每个新浏览器第一次编辑时仍要输入一次，这是正常的。

## 2. 地图白屏

旧版强制使用高德卫星图层：有些浏览器能加载控件，但卫星瓦片加载失败，就会出现“有 + - 按钮，但底图白屏”。

这版已改成默认高德矢量底图，不强制卫星图层，稳定性更高。点位、点击显示卡片、拖动点位等功能保留。

如果仍提示 Key 错误，请检查：

- Key 类型必须是高德「Web端 JS API」。
- 域名白名单包含 `hgcjlxz.pages.dev`。
- 如果开启安全密钥，`securityJsCode` 必须填对。
- 进入开发者权限后点“设置 Key”，保存后等待云端同步。

## 3. 之前图片为什么换浏览器不显示

没有 R2 的情况下，从电脑直接上传的图片只能保存在当前浏览器本地，不能跨浏览器同步。旧浏览器里能看到，不代表已经进了 D1 云端。

这版继续保护 D1：`data:image/base64` 不写入云端，避免把 `/api/content` 撑爆。要跨浏览器显示，请把图片放到仓库 `assets/images/`，然后在“管图”里添加静态路径，例如：

```text
assets/images/teaching/building-01.jpg
```

如果旧 Edge 里还能看到那些图片，可以在旧 Edge 里点“导出备份”，把备份 JSON 发给我，我可以帮你把里面的 base64 图片转成 `assets/images/` 静态文件。
