# 湖北工程学院新生导览网站：云端同步升级说明

这个版本已经加入 Cloudflare Pages Functions、D1、R2 的代码。

## 目标效果

- 访客打开网站即可浏览。
- 主人进入主人权限后编辑板块、卡片、Q&A。
- 主人上传图片后，图片保存到 Cloudflare R2。
- 文字内容和图片列表保存到 Cloudflare D1。
- 所有访客刷新后看到同一份最新内容。

## 必须使用 GitHub 或 Wrangler 部署

Cloudflare Pages 的“直接上传 zip”适合纯静态站，但不支持 Functions。
这个云端同步版需要 `functions/` 目录生效，所以要改用：

1. GitHub 仓库连接 Cloudflare Pages；或
2. Wrangler 命令行部署。

推荐 GitHub 仓库连接 Pages。

## Cloudflare 资源名称

在 Cloudflare 创建这些资源：

- D1 数据库：`hbeu_site`
- R2 bucket：`hbeu-images`
- Pages 环境变量：`ADMIN_TOKEN`

`ADMIN_TOKEN` 的值要和网站里第一次设置的“主人权限”口令一致。

## Pages 绑定名称

在 Cloudflare Pages 项目的 Settings 里绑定：

- D1 binding name：`DB`
- R2 binding name：`IMAGES`

绑定名必须完全一致，否则 API 会报错。

## 初始化 D1

把 `migrations/0001_site_state.sql` 执行到 D1 数据库。

SQL 内容：

```sql
CREATE TABLE IF NOT EXISTS site_state (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL,
  updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

## 部署后检查

打开：

```text
https://你的域名/api/content
```

如果看到 JSON，说明后台 API 生效。

正常第一次会类似：

```json
{"cloud":true,"content":null,"imageStacks":{},"updatedAt":null}
```

然后进入网站主人权限，改一次 Q&A 或卡片并保存。刷新其它设备，应该能看到同步后的内容。

## 注意

- 旧的 localStorage 仍保留为本地兜底。
- 云端没配置好时，网站仍能浏览，但不会全网同步。
- 图片仍会压缩并限制单槽数量，避免 R2 用量过快增长。
